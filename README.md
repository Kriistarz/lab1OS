# Задание 1 — sys-info-win
## Текст задачи
Научитесь работать с Windows API для получения системной информации, реализуйте структурированный и надёжный код, который:
Корректно обрабатывает все возможные ошибки,
Даёт студенту представление о том, как устроена система «под капотом».
Напишите программу sys-info-win для ОС Windows, которая выводила бы в консоль информацию о компьютере, на котором она запущена.
Версия операционной системы.
Используйте IsWindows10OrGreater() и аналогичные функции из VersionHelpers.h 
Размер виртуальной и физической памяти, а также использование памяти в процентах.
Количество ядер процессора
Имя компьютера и имя пользователя
Архитектура процессора (x86, x64, ARM)
Размер файла подкачки (функция GetPerformanceInfo)
Список логических дисков и их объёмы
## Алгоритм решения
1. Импортировать необходимые модули
Используются:
ctypes — для работы с WinAPI и C-структурами
pywin32 (win32api, win32file) — высокоуровневые обёртки WinAPI
platform, os, socket, getpass — стандартные модули
Это позволяет получать всю нужную системную информацию.
2. Определение версии операционной системы
Алгоритм:
Получить строковую версию Windows через platform.release().
Сравнить значение:
содержит "10" или "11" → Windows 10 or Greater
"8.1" → Windows 8.1
"8" → Windows 8
"7" → Windows 7
иначе → более старая версия
Вернуть название версии ОС.
3. Определение архитектуры процессора
Алгоритм:
Вызвать win32api.GetNativeSystemInfo().
Извлечь из структуры поле wProcessorArchitecture.
Сопоставить код:
9 → x64
0 → x86
12 → ARM64
иначе → Unknown
4. Получение имени компьютера
Алгоритм:
Выполнить win32api.GetComputerName().
Если ошибка — fallback: socket.gethostname().
5. Получение имени пользователя
Алгоритм:
Вызвать win32api.GetUserName().
Если ошибка — fallback: getpass.getuser()
6. Получение количества процессоров
Алгоритм:
Вызвать win32api.GetSystemInfo().
Из структуры взять значение — количество процессоров.
Если возникает ошибка — взять os.cpu_count().
7. Получение информации о памяти (RAM и Virtual Memory)
Алгоритм:
Создать объект структуры MEMORYSTATUSEX.
Перед вызовом WinAPI заполнить:
dwLength = размер структуры.
Вызвать WinAPI:
GlobalMemoryStatusEx().
Если возвращён 0 → ошибка.
Из структуры извлечь:
ullTotalPhys
ullAvailPhys
ullTotalPageFile
dwMemoryLoad
Перевести данные из байт в мегабайты.
Вывести:
полный объём RAM
использованный RAM
виртуальная память
процент загрузки памяти
8. Получение данных о файле подкачки
Алгоритм:
Создать структуру PERFORMANCE_INFORMATION.
Установить поле cb (размер структуры).
Вызвать WinAPI-функцию:
GetPerformanceInfo().
Если ошибка — вывести сообщение.
Извлечь:
CommitTotal
CommitLimit
PageSize
Рассчитать объёмы pagefile:
полный размер = CommitLimit * PageSize
занято = CommitTotal * PageSize
Перевести в МБ.
Вывести в консоль.
9. Получение списка логических дисков
Алгоритм:
Вызвать win32api.GetLogicalDriveStrings().
Разбить строку по нулевому символу.
Получить список дисков вида:
C:\
D:\
и т.д.
10. Получение информации о каждом диске
Алгоритм:
Для каждого диска:
Вызвать win32file.GetDiskFreeSpaceEx(disk).
Извлечь:
свободное пространство
общий объём диска
Перевести байты → ГБ.
Вывести в консоль.
Если возникла ошибка — вывести сообщение.
11. Формирование и вывод результата


# Задание 2 — sys-info-linux
## Текст задачи
Напишите программу sys-info-linux для ОС Linux, которая выводила бы в консоль информацию о компьютере, на котором она запущена.
Версия ядра и дистрибутива
Используйте uname() для ядра
Используйте lsb_release() или чтение /etc/os-release
Объём свободной и имеющейся оперативной памяти (в мегабайтах)
Используйте sysinfo или парсинг /proc/meminfo.
Количество логических процессоров (get_nprocs())
Архитектура процессора
Используйте uname().machine
Загрузка процессора (из sysinfo.loads) или /proc/loadavg.
Список подключенных логических дисков
прочитать /proc/mounts или вызвать getmntent.
Получить статистику через statvfs()
Информация о текущем пользователе и имя хоста
getlogin(), gethostname() или getpwuid(getuid()).
Объём доступной виртуальной памяти
Через /proc/meminfo, поле VmallocTotal (если доступно)
## Алгоритм решения
1. Получение информации о дистрибутиве Linux
Функция: get_os_info()
Открыть файл /etc/os-release.
Прочитать его построчно.
Для каждой строки вида KEY=VALUE:
разделить по =,
удалить кавычки,
сохранить в словарь.
Сформировать строку "NAME VERSION".
Если файл не найден → вернуть "Unknown".
2. Получение информации о памяти
Функция: get_memory_info()
Открыть файл /proc/meminfo.
Построчно прочитать пары "Ключ: Значение".
Сохранять числовое значение (кБ) в словарь.
После чтения вычислить:
MemTotal / 1024 → MB
MemAvailable / 1024 или MemFree / 1024
SwapTotal / 1024
SwapFree / 1024
VmallocTotal / 1024
Вернуть кортеж значений.
3. Получение загрузки процессора
Функция: get_load_average()
Открыть файл /proc/loadavg.
Прочитать первую строку.
Взять первые три числа.
Они соответствуют среднему load average за 1, 5 и 15 минут.
Вернуть их в виде списка строк.
4. Получение информации о дисках
Функция: get_drives_info()
Открыть /proc/mounts.
Разобрать каждую строку на:
устройство,
точку монтирования,
тип файловой системы.
Пропустить ФС, не входящие в список: ("ext4", "devpts").
Исключить повторяющиеся точки монтирования.
Для каждой точки:
вызвать os.statvfs();
вычислить:
total = blocks * frsize
free = bavail * frsize
перевести в гигабайты.
Сохранить всё в список.
Вернуть список дисков.
5. Сбор основной системной информации
В main():
os_info = get_os_info()
Получить версию ядра:platform.system() + platform.release()
Получить архитектуру CPU:platform.machine()
Получить имя хоста:socket.gethostname()
Получить пользователя:getpass.getuser()
Получить память:get_memory_info()
Получить загрузку CPU:get_load_average()
Количество логических ядер:os.cpu_count()
Получить список дисков:get_drives_info()
